## Goal

Spec and ship the **golden path** from `01-product-surface.md` as one owned, instrumented
funnel: stranger → free result → signup (domain carried over) → **first full audit free** →
first fix applied → delta shown. Every step exists to get the user to the "Fixed ✓ +score"
moment in their first session.

## The funnel

1. **Free public tools** (marketing site, no auth, cached + rate-limited, zero credits):
   - `…/free-tools/ai-visibility-check` — the V1.5 snapshot (already planned).
   - `…/free-tools/ai-crawler-checker` — V1.1 standalone ("which AI bots can read your site").
   - `…/free-tools/llms-txt-generator` — V1.3 standalone (real search volume, instant value).
   - `…/free-tools/what-ai-says-about-you` — V5.5 teaser: 1–2 fixed prompts for the domain,
     heavily cached, strictly rate-limited. The most visceral hook we have.
2. **Capture:** every result page offers "email me the full report" (email capture) and a
   signup CTA that carries the domain + a result token.
3. **Carry-over:** snapshot/teaser result stored in KV under a short-lived token; signup with
   the token pre-fills the brand website and shows "here's what we already found."
4. **First audit free:** onboarding auto-runs one full audit via a one-time
   `first_audit_free` workspace flag (a grant, not a `CREDIT_COSTS` bypass — the ledger still
   records it). Charging 50 credits before showing value is friction at the worst moment.
5. **First fix:** after the audit, route to the fix queue with the top `artifact` finding
   open — a copy-paste win inside minutes.
6. **Share:** opt-in public score badge (SVG + OG image per domain) linking back to the free
   checker — every proud customer is a referral.

## Files to create

- Marketing-site routes for the four free tools (reuse the public quick endpoint pattern:
  cached by domain, rate-limited, never authed, never metered).
- `src/app/api/public/teaser/route.ts` — the V5.5 one-prompt teaser (aggressive KV cache,
  per-IP + per-domain rate limits; this endpoint spends real engine-API money — cache hard).
- `src/lib/growth/handoff.ts` — result token: KV write on the public side, read + brand
  pre-fill during onboarding.
- Onboarding step: `first_audit_free` grant + auto-run + progress → land on `/visibility`.
- `src/app/api/public/badge/[domain]/route.ts` — score badge SVG/OG (only for workspaces
  that opted in).
- Funnel events (tool run → email captured → signup → audit done → first fix → badge shared)
  in whatever analytics the app already uses — the funnel must be measurable end-to-end.
