# V8.2 — Fix queue

- **Status:** ☐ Not started
- **Type:** ⚙️ UI/plumbing (no LLM)
- **Depends on:** [V8.1](v8.1-visibility-dashboard.md), findings with `fix_capability` +
  `fix_payload` (engine contract #1, `00-principles.md` rule 6)
- **Unlocks:** V7.2 gets its UI, V8.5 approval queue reuses this surface

## Goal

Ship `/visibility/fixes` — ONE severity-ranked queue merging every finding from every analyzer
(full audits, Toolbox runs, agent runs). Each row has exactly one primary action driven by
`fix_capability`. This queue **is** the product's interface; tools are just where rows come from.

## Files to create

- `src/app/(app)/visibility/fixes/page.tsx` — shell + queue section.
- `src/components/visibility/fix-queue.tsx` — the ranked list: severity groups
  (Critical → High → Medium → Low), pillar badge, title, plain-English impact line.
- `src/components/visibility/fix-action.tsx` — renders the row's button by capability:
  - `auto` → **Fix it for me** (calls V7.2 apply endpoint; disabled with tooltip until V7.2 ships)
  - `artifact` → **Get the fix** (drawer rendering `fix_payload` as copy-paste code: JSON-LD,
    robots.txt block, meta tags, llms.txt)
  - `guided` → **Show me how** (drawer with the finding's step-by-step `recommendation`)
- `src/app/api/visibility/findings/route.ts` — GET open findings (filter by pillar/severity),
  PATCH dismiss/complete.
- `src/lib/visibility/findings-repository.ts` — query + resolution logic shared with V7.3.

## Build steps

1. Query `audit_findings` where `is_resolved = false`, deduped across runs by
   (`category`, `title`, page URL) — the same issue found by an audit and a Toolbox run is one row.
2. Row anatomy: severity chip, pillar badge (🔵🟣🟢), title, one-sentence impact in owner
   language (no jargon — "AI assistants can't read this page", not "SSR failure").
3. Wire the three action types. `artifact` drawers must have a copy button and say where to
   paste it. Until V7.2 exists, `auto` rows fall back to the `artifact` presentation with a
   "soon Claudia can apply this for you" note.
4. Dismiss (won't fix, hides) and manual complete (I did it, pending verification) actions.
5. Auto-resolution: when a newer audit no longer reports a finding, flip `is_resolved` and
   surface it in a collapsed "Fixed ✓" section with the score delta since — the proof moment.
6. Filters: pillar, severity, capability. Default view = everything, ranked.

## Acceptance

- [ ] Findings from a full audit and from a standalone Toolbox run appear in the same queue,
      deduplicated.
- [ ] Every row has exactly one working primary action matched to its `fix_capability`.
- [ ] Artifact drawer renders valid copy-paste output from `fix_payload` for at least:
      robots.txt (V1.1), meta tags (V1.4), llms.txt (V1.3), JSON-LD (V3.3).
- [ ] Re-running an audit auto-resolves fixed findings and shows them under "Fixed ✓".
- [ ] Empty state (no findings / no audit yet) links to running an audit.
