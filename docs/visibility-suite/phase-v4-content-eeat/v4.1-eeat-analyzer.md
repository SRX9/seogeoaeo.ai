## Goal

Score **Experience, Expertise, Authoritativeness, Trustworthiness** (0–25 each) from observable
signals, with evidence per dimension.

## Files to create

- `src/lib/visibility/content.ts` — `analyzeEeat(snapshot): EeatResult` (this ticket's slice).
- `src/lib/visibility/eeat-schema.ts` — Zod schema for the LLM JSON output.
- fixture test of the parser/aggregator.

## Build steps

1. Build a `heavy`-tier prompt that ingests extracted content (headings, paragraphs, author byline,
   dates, contact info, external links) and the **exact signal checklists** per dimension.
2. Require JSON out: `{ experience:{score,evidence[]}, expertise:{…}, authoritativeness:{…},
   trustworthiness:{…} }`, each `score` 0–25. Validate with Zod; reject + retry on malformed.
3. Apply the 0–25 banding rubric per dimension (0–5 none … 21–25 exceptional).
4. Normalize each 0–25 → 0–15 for the content-score weighting (done in V4 aggregation).
5. Emit findings (e.g., "No author credentials — add bylines + author page + Person schema").
6. Test: a fixture article with author+citations+HTTPS scores high; an anonymous thin page low.
