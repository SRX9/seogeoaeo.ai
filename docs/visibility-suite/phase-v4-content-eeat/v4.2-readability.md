## Goal

Word-count classification, approximate Flesch reading ease, paragraph-length flags, and
heading-hierarchy validation — all deterministic so it can run live in the editor.

## Build steps

1. **Word count tiers:** `<300` thin · 300–800 short · 800–1500 standard · 1500–3000 long · 3000+
   deep-dive.
2. **Flesch:** sample 3–5 representative paragraphs; `206.835 − 1.015·(avg words/sentence) −
   84.6·(avg syllables/word)`; map to level; note "approximation from sampling". Ideal 50–70.
3. **Paragraph length:** avg words; flag >150 ("wall of text"); ideal 40–80.
4. **Heading hierarchy:** exactly one H1, no skipped levels, descriptive, density ~1 H2/H3 per
   200–300 words (from the V0.1 heading tree).
5. Emit metrics + findings. Tests: thin page flagged; skipped-heading-level detected; Flesch within
   tolerance on a known sample.
