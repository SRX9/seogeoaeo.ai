## Goal

Assess content breadth, internal-link depth, and hub/cluster structure; output a **content-gap
list** that feeds the topic backlog.

## Files to create

- add `analyzeTopicalAuthority(snapshot, siteUrls): TopicalAuthorityResult` to `content.ts`.
- test.

## Build steps

1. **Deterministic:** count internal links on the page + across discovered URLs (V0.2); detect
   hub-and-spoke / pillar-cluster structure; measure breadth (related pages on the topic).
2. **Topical-authority modifier (port):** +10 for 20+ pages with strong clustering, scaling down to
   **−5** for fewer than 5 pages on the topic.
3. **LLM `light` gap synthesis:** given the topic + observed pages, list notable missing subtopics.
4. Write gaps to the topic backlog (reuse content-roadmap Phase 4 research records) so our writer
   can fill them — closing the optimize→create loop.
5. Test: a deep clustered fixture → +modifier; a thin 3-page site → −5 + gap list.
