# V4.5 — Content freshness tracker

- **Status:** ☐ Not started
- **Type:** ⚙️ Deterministic
- **Pillar:** 🔵 SEO · Catalog #13
- **Depends on:** [V0.1](../phase-v0-foundation/v0.1-page-fetcher.md)
- **Unlocks:** V2.3 (freshness component), V7.3 (auto-refresh suggestions)

## Goal

Detect publish/updated dates, flag stale time-sensitive (YMYL) content, and surface refresh
candidates for the scheduler.

## Files to create

- add `analyzeFreshness(snapshot): FreshnessResult` to `content.ts`.
- test.

## Build steps

1. Extract publication + last-modified dates (meta, Article schema, visible "Updated…" text).
2. Compute content age; detect time-sensitivity (news/stats/tech = high; evergreen = low).
3. **Flag content older than 2 years on time-sensitive topics**; weight YMYL (health/finance/legal/
   safety) higher.
4. Emit refresh candidates (page + reason) → feed V7.3 / the scheduler for auto-refresh suggestions.
5. Test: a 2023-dated tech fixture → stale flag; an evergreen page → no flag.

## 📐 Build from (source of truth)

`inspiration-code/agents/geo-content.md` → **Step 9** "Content Freshness" (date signals,
time-sensitivity, the >2-years flag) and "Important Notes" (YMYL weighting).
