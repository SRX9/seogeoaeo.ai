# V3.3 — JSON-LD generator (incl. FAQ / Speakable)

- **Status:** ☐ Not started
- **Type:** 🧠 LLM + deterministic templates
- **Pillar:** 🔵 SEO / 🟣 AEO · Catalog #9, #16
- **Depends on:** [V3.2](v3.2-schema-score-sameas.md), [V0.4](../phase-v0-foundation/v0.4-business-type-detector.md)
- **Unlocks:** V7.2 (auto-insert), reports

## Goal

Generate ready-to-paste, business-type-aware JSON-LD for every missing/incomplete schema —
including FAQPage/QAPage and `speakable` — stored as `fix_payload`.

## Files to create

- `src/lib/visibility/schema/generate.ts` — `generateSchema(gaps, businessType, snapshot): SchemaFix[]`.
- `src/lib/visibility/schema/templates/` — TS template literals ported from the 6 JSON files.
- tests (valid JSON, required props present, placeholders correct).

## Build steps

1. Port the **6 reference templates** to typed template builders: Organization (+ comprehensive
   sameAs + knowsAbout), LocalBusiness, Article+Author(Person)+speakable, Product+Offer+Review,
   SoftwareApplication, WebSite+SearchAction.
2. From V3.2 gaps + business type, pick which templates to emit (Local → LocalBusiness; SaaS →
   SoftwareApplication; Publisher → Article+Person; all → Organization+sameAs, WebSite+SearchAction).
3. Fill known values from the snapshot (name, url, logo, author); leave the rest as
   `[REPLACE: description]` placeholders. Always JSON-LD, `@context: https://schema.org`, `@graph`
   where multiple types.
4. Generate **FAQPage/QAPage + speakable** from page Q&A content when relevant (AEO answer surfaces).
5. Validate generated JSON parses + has required props; store each as a `fix_payload` on the finding.
6. Tests: generated Organization parses + has sameAs; Article has Person author + dateModified +
   speakable; placeholders use the `[REPLACE: …]` convention.

## 📐 Build from (source of truth)

- `inspiration-code/schema/*.json` — the **6 templates**: `organization.json`, `local-business.json`,
  `article-author.json`, `product-ecommerce.json`, `software-saas.json`, `website-searchaction.json`.
- `inspiration-code/agents/geo-schema.md` → **Step 7** (generation rules, required+recommended props,
  `[REPLACE: …]` placeholder convention) and "Important Notes" (JSON-LD only; speakable underused;
  FAQ not harmful on non-authority sites).
