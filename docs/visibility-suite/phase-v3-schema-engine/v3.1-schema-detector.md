## Goal

Extract all JSON-LD / Microdata / RDFa, validate types + required properties, check Google
rich-result eligibility, and flag deprecated + JS-injected schema.

## Files to create

- `src/lib/visibility/schema/detect.ts` — `detectSchema(snapshot): SchemaBlock[]` (deterministic).
- `src/lib/visibility/schema/validate.ts` — `validateSchema(blocks): ValidatedSchema[]` (rules +
  LLM `light` for semantic/rich-result judgement).
- tests for both.

## Build steps

1. **Detect (deterministic):** JSON-LD already in `snapshot.structured_data`. Also scan HTML for
   Microdata (`itemscope`/`itemtype`/`itemprop`) and RDFa (`vocab`/`typeof`/`property`). Record
   count, format(s), and all `@type`s.
2. **Validate:** `@context` = schema.org, recognized `@type`, valid property names, ISO-8601 dates,
   absolute URLs, proper nesting (author as Person object, not string). Flag common errors
   (missing `@context`, misspelled props, wrong value types, placeholder values, dup blocks).
3. **Rich-result eligibility:** map each type to Google's requirements table; note missing required
   + recommended props.
4. **Deprecated/restricted flags:** HowTo (removed Sep-2023), FAQPage (restricted Aug-2023 to
   gov/health), SpecialAnnouncement/CourseInfo (deprecated).
5. **JS-injection risk:** if the site is a JS framework and schema isn't in the raw HTML, flag it
   (AI crawlers won't execute JS → schema invisible). Reuse V0.1 SSR signals.
6. Persist `schema_blocks` rows. Tests: valid Organization, an Article with string author (error),
   a HowTo (deprecated flag), JS-injected case.
