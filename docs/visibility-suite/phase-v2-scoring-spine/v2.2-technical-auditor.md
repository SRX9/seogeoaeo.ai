## Build steps — port the agent's 11 steps

1. **Headers/status** (from V0.1): status, redirects, content-type, cache, compression, `X-Robots-Tag`.
2. **Crawlability**: robots validity + AI crawler access (reuse V1.1) + sitemap presence/validity.
3. **Meta/indexability**: reuse V1.4; add canonical/duplicate (www/HTTP/trailing-slash)/hreflang.
4. **Security headers (deductions, port exactly):** no HTTPS **−30**, no HSTS −10, no CSP −10,
   no X-Frame-Options −5, no X-Content-Type-Options −5, no Referrer-Policy −5, no Permissions-Policy −3.
5. **URL structure**: clean/readable/hierarchical → 80–100 … problematic → 0–39 (lowercase, hyphens,
   depth ≤4, length <100).
6. **Mobile**: viewport, media queries, responsive images (`srcset`/`<picture>`), tap targets, ≥16px font.
7. **CWV risk (static-HTML estimate — label it):** LCP/INP/CLS risk indicators (lazy/ fetchpriority,
   render-blocking, font-display, image dims, heavy sync JS). Use **INP, never FID**.
8. **SSR (highest weight):** reuse V0.1 `has_ssr_content`; classify severity CRITICAL/HIGH/MEDIUM/LOW
   + detect framework (`__NEXT_DATA__`/`__NUXT__`/`data-server-rendered`/SPA shells).
9. **Additional checks**: redirect chains, hreflang, JSON-LD syntax, resource hints.
10. (V5.3 adds agent-readiness here — leave a hook.)
11. **Score:** weights — SSR 25%, Meta 15%, Crawlability 15%, Security 10%, CWV 10%, Mobile 10%,
    URL 5%, Response 5%, Additional 5%.
