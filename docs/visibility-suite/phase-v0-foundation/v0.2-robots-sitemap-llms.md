## Goal

Fetch + parse `robots.txt` (per-user-agent rules, sitemaps, AI-crawler status), crawl sitemaps to
discover ‚â§50 pages, probe for `llms.txt`/`llms-full.txt`, and split a page into heading-bounded
content blocks (consumed by the citability scorer).

## Build steps

1. **robots.ts:** GET `/robots.txt`. Parse line-by-line tracking the current `User-agent`; collect
   `Disallow`/`Allow` per agent and `Sitemap:` lines (fix the `http` split-off bug). For each of
   the 14 AI crawlers, classify: `BLOCKED` (Disallow `/`), `PARTIALLY_BLOCKED`, `ALLOWED`,
   `BLOCKED_BY_WILDCARD`, `ALLOWED_BY_DEFAULT`, `NOT_MENTIONED`, `NO_ROBOTS_TXT`. Return raw text too
   (V1.2 scans it for `Content-Signal:`).
2. **sitemap.ts:** try `/sitemap.xml`, `/sitemap_index.xml`, `/sitemap/`; handle sitemap-index
   recursion (fetch child `<loc>`s); dedupe; cap at `max`.
3. **llms.ts:** GET `/llms.txt` and `/llms-full.txt`; record existence + raw content (validation is
   V1.3).
4. **blocks.ts:** strip `script,style,nav,footer,header,aside`; walk `h1‚Äìh6,p,ul,ol,table,blockquote`;
   start a new block at each heading; emit `{heading, content, word_count}`; drop blocks `<20` words.
5. Tests against fixtures: a robots.txt that blocks GPTBot, one with `Disallow: /` wildcard, a
   sitemap index, and an article page (block count + headings).
