- **Status:** ☑ Done
- **Type:** ⚙️ Deterministic (+ 🧠 fallback)

- **Unlocks:** tailored recommendations + schema choices in V3.3, V5, reports

## Goal

Classify a site as **SaaS / Local / E-commerce / Publisher / Agency / Other** so every downstream
recommendation and generated schema is tailored.

## Files to create

- `src/lib/visibility/business-type.ts` — `detectBusinessType(snapshot): BusinessType`.
- `src/lib/visibility/business-type.test.ts`.

## Build steps

1. Deterministic signal scan over the homepage `PageSnapshot` using the signal table below.
2. Score each type by matched signals; return the top match + confidence.
3. If confidence is low/ambiguous, fall back to a single LLM `light` classification call
   (title + meta + nav text → one of the 6 labels) — keep it optional/cheap.
4. Export a `recommendationProfile(type)` helper other tickets read (e.g., Local → push
   LocalBusiness schema + GBP; SaaS → SoftwareApplication + comparison pages).

## 📐 Build from (source of truth)

`inspiration-code/geo/SKILL.md` → **"Business Type Detection"** table — port the exact signal lists:
- **SaaS:** pricing page, "Sign up", "Free trial", `/app`, `/dashboard`, API docs.
- **Local:** phone, address, "Near me", Maps embed, service area.
- **E-commerce:** product pages, cart, "Add to cart", price elements, Product schema.
- **Publisher:** blog, articles, bylines, publication dates, Article schema.
- **Agency:** portfolio, case studies, "Our services", client logos, testimonials.
- **Other:** default — general GEO best practices.

Plus the "adjust recommendations based on detected type" guidance in the same section.
