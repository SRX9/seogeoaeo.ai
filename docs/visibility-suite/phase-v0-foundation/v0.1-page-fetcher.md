## Goal

Given a URL, return one typed `PageSnapshot`: status, redirect chain, response + security headers,
meta tags, canonical, full heading tree, word count, internal/external links, images
(alt/dimensions/loading), parsed JSON-LD blocks, and an SSR flag.

## Files to create

- `src/lib/visibility/types.ts` — shared types (start here; other tickets extend it).
- `src/lib/visibility/fetch-page.ts` — `fetchPage(url, opts?): Promise<PageSnapshot>`.
- `src/lib/visibility/fetch-page.test.ts` — fixture tests.
- `test/fixtures/visibility/` — saved HTML fixtures (SSR page, CSR SPA shell, WordPress page).

## Build steps

1. Define `PageSnapshot` in `types.ts` mirroring the Python `result` dict (see source below).
2. Implement `fetchPage` with Workers `fetch()`: follow redirects, record the chain, capture all
   response headers, enforce a 30s timeout (`AbortController`), and reject non-`http(s)` schemes.
3. Parse the **raw** HTML with `linkedom`. Extract title, meta tags (`name`/`property` → content),
   canonical, heading tree (h1–h6 with level + text), images, internal vs external links.
4. **Extract JSON-LD before any DOM mutation** — collect every
   `<script type="application/ld+json">`, `JSON.parse` each, push parse errors to `errors[]`.
5. **SSR heuristic (port exactly):** find framework root elements matching `id=/app|root|__next|__nuxt/i`,
   measure their inner text length *before* stripping nodes. After measuring, decompose
   `script,style,nav,footer,header` and compute `word_count`. Flag `has_ssr_content = false` only
   when a root has `<50` chars **and** total `word_count < 200` (avoids false positives on
   SSR/cached sites like WordPress).
6. Capture the six security headers into `security_headers`.
7. Tests: assert SSR page → `has_ssr_content true`; CSR shell fixture → `false`; JSON-LD count
   correct; redirect chain captured.
