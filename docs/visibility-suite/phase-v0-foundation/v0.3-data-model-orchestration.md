# V0.3 — Audit data model & orchestration Workflow

- **Status:** ☐ Not started
- **Type:** ⚙️ Deterministic
- **Pillar:** ⚪ Foundation
- **Depends on:** [V0.1](v0.1-page-fetcher.md), [V0.2](v0.2-robots-sitemap-llms.md)
- **Unlocks:** V2.3 and all persistence

## Goal

Drizzle tables + a Cloudflare Workflow that runs an audit end-to-end (Discovery → Analysis →
Synthesis) and persists results. Analyzers are stubbed now; later phases fill them in.

## Files to create

- `src/db/schema/visibility.ts` — Drizzle tables (see data-model in principles).
- `drizzle/` migration (via `pnpm db:generate`).
- `src/server/visibility/run-audit.ts` — the Workflow / orchestrator `runAudit(workspaceId, siteUrl)`.
- `src/server/visibility/analyzers.ts` — analyzer registry (stub functions returning empty subscores).
- `src/app/api/visibility/audit/route.ts` — kick off an audit, return `auditId`.

## Build steps

1. Create the tables from [`../00-principles.md`](../00-principles.md) → "Shared data model":
   `audits`, `audit_pages`, `audit_findings`, `citability_blocks`, `schema_blocks`,
   `brand_signals`, `platform_scores`. Add `run_version` for delta tracking (V6.3).
2. **Discovery stage:** `fetchPage(homepage)` (V0.1), `fetchRobots` + `crawlSitemap` + `fetchLlmsTxt`
   (V0.2), detect business type (V0.4), store the homepage `audit_page` + up to 50 discovered URLs.
3. **Analysis stage:** `Promise.all` over the analyzer registry (mirrors the 5 parallel subagents).
   Each analyzer takes the snapshot(s) and returns `{ subScore, findings[], rows[] }`. Stubs for now.
4. **Synthesis stage:** compute the composite (V2.3 fills the formula), persist sub-scores +
   findings, set `status = complete`.
5. Enforce quality gates (≤50 pages, ≤5 concurrent, ~1s spacing, 30s/page) in the fan-out.
6. API route starts the Workflow and returns `auditId`; a `GET` returns status + result.
