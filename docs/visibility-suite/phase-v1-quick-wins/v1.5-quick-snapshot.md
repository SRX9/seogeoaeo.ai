# V1.5 — 60-second quick snapshot

- **Status:** ☐ Not started
- **Type:** ⚙️ Deterministic
- **Pillar:** ⚪ Cross-cutting · Catalog #2
- **Depends on:** [V1.1](v1.1-crawler-access.md), [V1.2](v1.2-content-signals.md), [V1.3](v1.3-llms-txt.md), [V1.4](v1.4-meta-audit.md), [V2.1](../phase-v2-scoring-spine/v2.1-citability-scorer.md) (single-block read)
- **Unlocks:** public lead-gen tool, prospect/CRM flow

## Goal

A lightweight pass — crawler access, llms.txt presence, homepage schema presence, a rough
citability read on hero content — returning an approximate score + top gaps. **No full crawl.**

## Files to create

- `src/app/api/visibility/quick/route.ts` — `POST { url }` → snapshot result.
- `src/lib/visibility/quick.ts` — `quickSnapshot(url): QuickResult`.
- a public UI entry (landing "check your site" form) wired to the route.

## Build steps

1. `fetchPage(homepage)` + `fetchRobots` + `fetchLlmsTxt` (V0.1/V0.2) — one homepage fetch only.
2. Run V1.1 (crawler access), V1.2 (content signals), V1.3 validate-only, V1.4 (meta), homepage
   schema presence (count JSON-LD blocks), and a single citability read on the hero block (V2.1).
3. Combine into an **approximate** score (lightweight weighting of the available signals; label it
   "estimate — run full audit for the real score") + top 3–5 gaps.
4. **Cache by domain** (KV, short TTL) and **rate-limit** the public route (abuse protection).
5. When called from the prospect/CRM flow (V7.4), persist the score to the prospect record.

## 📐 Build from (source of truth)

`inspiration-code/docs/commands-reference.md` → **"/geo quick"** (the exact lightweight signal set,
"no file written", and that it feeds `/geo prospect audit`). Composes V1.1–V1.4 + V2.1.
