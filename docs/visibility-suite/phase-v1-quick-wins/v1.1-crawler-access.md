# V1.1 — AI crawler access analyzer

- **Status:** ☑ Done
- **Type:** ⚙️ Deterministic
- **Pillar:** 🟢 GEO · Catalog #20
- **Depends on:** [V0.2](../phase-v0-foundation/v0.2-robots-sitemap-llms.md)
- **Unlocks:** V1.5, V2.3 (feeds AI-visibility sub-score)

## Goal

Map robots.txt against 14+ AI crawlers in 3 tiers, compute a deterministic AI-visibility score,
and output a ready-to-paste robots.txt configured for maximum AI visibility.

## Files to create

- `src/lib/visibility/crawler-access.ts` — `analyzeCrawlerAccess(robots): CrawlerAccessResult`.
- `src/lib/visibility/crawler-access.test.ts`.

## Build steps

1. Consume the `RobotsResult` from V0.2 (don't re-fetch).
2. Group crawlers into tiers: **Tier 1** (GPTBot, OAI-SearchBot, ChatGPT-User, ClaudeBot,
   PerplexityBot), **Tier 2** (Google-Extended, GoogleOther, Applebot-Extended, Amazonbot,
   FacebookBot), **Tier 3** (CCBot, anthropic-ai, Bytespider, cohere-ai).
3. **Score (port exactly):** start at 100; **−15 per critical crawler blocked** (GPTBot, ClaudeBot,
   PerplexityBot, OAI-SearchBot, GoogleBot); **−5 per secondary crawler blocked**; **−10 if no
   sitemap referenced**; floor 0.
4. Emit findings (severity by tier) + a `fix_payload` = the recommended robots.txt block (allow all
   Tier 1/2, keep sitemap line). Store for V7.2.
5. Tests: fully-open robots → 100; GPTBot blocked → 85; `Disallow: /` wildcard → low; no sitemap → −10.
