## Goal

Detect the IETF `Content-Signal:` directive in robots.txt, parse + validate it, explain it in
plain English, and recommend one if absent. **Never penalizes** (it's an IETF draft).

## Files to create

- extend `src/lib/visibility/crawler-access.ts` with `parseContentSignals(robotsText)`.
- add cases to `crawler-access.test.ts`.

## Build steps

1. Scan the **already-fetched** robots.txt text (zero extra requests) for `Content-Signal:` lines.
2. Parse comma-separated `key=value` pairs. Valid keys: `ai-train`, `search`, `ai-personalization`,
   `ai-retrieval`. Valid values: `yes` / `no`.
3. Classify: **present+valid** → pass (report parsed values + plain-English meaning); **unknown key
   or invalid value** → warning (flag the issue, suggest correction — draft is still evolving);
   **absent** → recommendation (explain what to add, link contentsignals.org).
4. Return a non-scoring `contentSignals` block on the crawler-access result (does **not** change the
   Crawler Access Score).
5. Tests: valid 3-pair directive → pass; `ai-input=yes` (unknown key) → warning; absent → recommendation.
