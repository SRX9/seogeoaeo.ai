## Goal

Validate an existing `llms.txt` against the spec, **or** generate a complete one (concise +
`llms-full.txt`) by crawling the site. Score it 0/30/50/70/90–100.

## Files to create

- `src/lib/visibility/llms.ts` — add `validateLlmsTxt(url)` and `generateLlmsTxt(url, max=30)`.
- `src/lib/visibility/llms.test.ts`.

## Build steps

1. **Validate:** GET `/llms.txt`. Check H1 title (`# Name`), blockquote description (`> …`),
   `##` sections, and link lines `- [Title](url)`. Compute `format_valid` = all four present.
   Build the suggestion list (`<5` links → add pages; `<2` sections; no contact; no key facts).
   Probe `/llms-full.txt`. Score by band (below).
2. **Generate:** fetch homepage, derive site name (title before `|`/`-`) + description (meta).
   Crawl internal links; bucket into **Main Pages / Products & Services / Resources & Blog /
   Company / Support** using the path-keyword map; skip assets + cross-origin. Emit concise
   `llms.txt` (≤10 links/section) + `llms-full.txt` (fetch each page's meta description; **skip
   cross-origin URLs to avoid SSRF via redirects**). Append a Contact section.
3. Store generated files as a `fix_payload` for V7.2.
4. **Score bands** (from agent): 0 absent · 30 malformed · 50 valid+minimal · 70 valid+covers
   primary areas · 90–100 comprehensive + `llms-full.txt`.
5. Tests: valid file → format_valid true + correct band; malformed → 30; generation buckets a
   fixture sitemap correctly + skips cross-origin.
