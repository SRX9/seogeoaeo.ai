## Goal

Score brand presence on the platforms LLMs learn entities from — Wikipedia/Wikidata, Reddit,
YouTube, LinkedIn, plus G2/Trustpilot/Capterra/Quora/SO/GitHub/HN/press — and return a weighted
Brand Authority Score with prioritized actions.

## Build steps

1. **Wikipedia (deterministic, authoritative — do this first):** call
   `en.wikipedia.org/w/api.php?action=query&list=search&srsearch=<brand>&format=json`; treat as
   found only if the top result title contains the brand. **Don't rely on web search** (false
   negatives). This is the single strongest entity signal.
2. **Wikidata:** `wikidata.org/w/api.php?action=wbsearchentities&search=<brand>&language=en&format=json`;
   capture the Q-id + description if present.
3. **Other platforms** (LLM/web-search-assisted, with the platform search URLs as evidence): Reddit
   (presence/recency/sentiment), YouTube (official channel + third-party videos), LinkedIn (company
   page), and G2/Trustpilot/Capterra/Quora/SO/GitHub/ProductHunt/Crunchbase.
4. **Score (port weights):** Wikipedia 30 · Reddit 20 · YouTube 15 · LinkedIn 10 · industry/niche 25
   → normalize to 0–100. (Platform display weights: YouTube 25% / Reddit 25% / Wikipedia 20% /
   LinkedIn 15% / Other 15%.)
5. Surface the **Ahrefs correlation** in the UI as the "why" (YouTube 0.737 vs domain rating 0.266).
6. Emit prioritized recommendations by time horizon. Persist `brand_signals` rows.
7. Test: a known brand with a Wikipedia page → high; an unknown brand → near 0.
