## Goal

Stop self-grading and measure the actual outcome. For a set of **tracked prompts** ("best
CRM for solo founders", "how do I do X"), ask the real engines and record whether the brand —
and its competitors — appear in the answer. The output is **answer share**: "you appeared in
4 of 10 tracked answers on Perplexity this week, up from 2." Tracked per engine, over time.

This is the single most persuasive number in the product. Audit scores say "you could be
cited"; answer share says "you *are* (or aren't) being cited." Every other tool in the suite
explains *why* this number is what it is and how to move it.

## How it works (implementable from a Worker)

Query each engine through its public API with web search/grounding enabled — no scraping:

| Engine | How |
|---|---|
| ChatGPT (web search) | OpenAI Responses API with the `web_search` tool |
| Perplexity | Perplexity Sonar API (returns citations natively) |
| Gemini | Gemini API with Google Search grounding |

Parse each answer for (a) **brand mention** — name-variant match in the answer text,
(b) **brand citation** — the brand's domain in the returned sources, (c) the same two checks
for each named competitor. Store everything; share-of-answer is computed from stored runs.

- `src/app/(app)/visibility/answers/page.tsx` — the **AI answers** page: share per engine
  with trend, and a prompt × engine grid where each cell is ✓ cited · ✓ mentioned ·
  ✗ absent · ⚠ competitor named instead.
- Scheduled runs: extend the existing daily agent job (V8.5 cadence logic) — answer checks
  run at the plan's cadence with the plan's prompt cap.

## Build steps

1. Schema + prompt suggestions; onboarding/settings UI to confirm the tracked prompt list
   (plan caps the count — see `01-product-surface.md` plans table).
2. Engine adapters with a shared `askEngine(prompt) → { text, citations[] }` contract;
   graceful per-engine failure (one engine down ≠ run failed).
3. Mention/citation detection: brand name variants (from brand profile) + apex-domain match;
   same for competitors. Deterministic and unit-tested — no LLM needed to detect a mention.
4. Persist `answer_runs`; compute share-of-answer per engine and delta vs the previous run.
5. Page + endpoints per the section-loading convention; on-demand runs metered, scheduled
   runs from plan allowance.
6. Emit findings for misses where we know the fix ("ChatGPT cites competitor X's comparison
   page — you don't have one") into `audit_findings` (`guided`, linked to the prompt) so the
   fix queue sees them.
