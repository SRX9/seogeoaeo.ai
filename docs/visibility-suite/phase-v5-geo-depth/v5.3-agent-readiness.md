## Build steps

1. **RFC 8288 Link headers (zero extra request):** parse `Link:` headers captured in V0.1. Identify
   high-value rel types: `api-catalog` (RFC 9609), `describedby`, `service-doc`, `mcp-server-card`.
   - present → document what was found.
   - absent **and** API-first site (API docs in nav, `/api/` or `/developers/`, OpenAPI in sitemap)
     → recommendation. Absent on a standard business site → **omit entirely**.
2. **Markdown content negotiation (one extra request):** GET the homepage with `Accept: text/markdown`.
   - response `Content-Type: text/markdown` → pass (leading-edge capability).
   - standard HTML → forward-looking recommendation (note Cloudflare one-line config).
   - error/non-200 → skip + note; **do not penalize**.
3. Return a non-scoring `agentReadiness` block under the technical result. Does **not** affect any score.
4. Test: a `Link:` header with `api-catalog` documented; markdown-negotiation pass/fail/error paths.
