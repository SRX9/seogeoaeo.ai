- **Status:** ☐ Not started
- **Type:** 🧠 + ⚙️ integration
- **Pillar:** 🟣 AEO / 🔵 SEO · Catalog #14, #11, #16, #19
- **Depends on:** [V2.1](../phase-v2-scoring-spine/v2.1-citability-scorer.md), [V4.2](../phase-v4-content-eeat/v4.2-readability.md), [V4.3](../phase-v4-content-eeat/v4.3-ai-content-detector.md), [V3.3](../phase-v3-schema-engine/v3.3-jsonld-generator.md), content-roadmap Phase 3 editor
- **Unlocks:** our moat — write content that's already optimized

## Goal

As the user (or our AI writer) drafts, show **live** citability, readability, and AI-content scores,
and auto-attach FAQ/Speakable/Article schema — so we *write optimized content*, not audit it after.

## Files to create

- `src/app/api/visibility/score-draft/route.ts` — `POST { markdown }` → live scores (debounced).
- editor UI panel (content-roadmap Phase 3) showing scores + inline suggestions.

## Build steps

1. On debounce, run the **deterministic** V2.1 citability + V4.2 readability on the draft (fast, no
   LLM) and render per-section scores + the weakest blocks.
2. On demand (button), run V4.3 AI-content check + V6.5 answer-block rewrite suggestions.
3. Auto-generate Article+Person+speakable (+ FAQ if Q&A present) via V3.3 and attach to the draft's
   front-matter / publish payload.
4. Ensure the **editor score == audit score** (same V2.1 module) so numbers are consistent.
5. Gate by plan if needed; keep the deterministic path free/cheap.

## 📐 Build from (source of truth)

Reuse V2.1 / V4.2 / V4.3 / V3.3 modules unchanged. The "live score" target is the citability rubric
in `inspiration-code/scripts/citability_scorer.py` (editor and audit must show identical numbers).

## Acceptance

- [ ] Draft shows live citability + readability scores (deterministic, debounced).
- [ ] On-demand AI-content check + answer-block rewrite suggestions.
- [ ] Article/Person/speakable/FAQ schema auto-attached to the draft.
- [ ] Editor score equals audit score for the same content.
