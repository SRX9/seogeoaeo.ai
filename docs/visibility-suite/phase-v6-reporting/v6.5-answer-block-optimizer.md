## Goal

Rewrite a target section into the shape answer engines extract — a **40–60 word direct answer up
top**, then list/table/steps — and **only ship it if citability improves**.

## Files to create

- `src/lib/visibility/fixes.ts` — `optimizeAnswerBlock(block): AnswerFix` (shared with V7.2).
- test.

## Build steps

1. Detect the snippet shape the query deserves (paragraph vs list vs table).
2. LLM `heavy` rewrite: lead with a concise 40–60 word direct answer, then structured detail; keep
   facts/citations intact.
3. **Score before + after with V2.1 citability.** Only return the rewrite as a `fix_payload` if the
   score increased; otherwise keep the original + explain.
4. Store the fix for one-click apply (V7.2) / live editor use (V7.1).
5. Test: a weak block fixture → rewrite with a higher citability score; a strong block → no-op.
