## Goal

Close the loop the north star promises. The score delta (V6.3) and answer share (V5.5) are
*our* numbers; this ticket brings in *their* numbers: *Google Search Console*
clicks/impressions and **AI-assistant referral traffic** from their analytics — rendered next
to the score trend so cause and effect sit on one chart. Never metered: **proof is free.**

## Scope

1. **Google Search Console (primary).** OAuth connect → verify the site property → pull
   daily clicks, impressions, average position into `traffic_snapshots`. This alone delivers
   "your score went 61→74 and clicks went +23%."
2. **GA4 AI referrals (secondary, optional).** OAuth connect → pull sessions where the
   referrer is an AI surface (`chatgpt.com`, `perplexity.ai`, `gemini.google.com`,
   `copilot.microsoft.com`, `claude.ai` — keep the list in one module) → store per-engine
   referral counts. This is the GEO payoff made visible.
3. **Out of scope:** our own tracking snippet (deferred — see `01-product-surface.md`),
   Bing Webmaster, position tracking per keyword.

## Files to create

- `src/lib/integrations/gsc.ts` — OAuth flow + Search Analytics API pull (site-level daily
  rows); token storage via the existing encrypted integration-secret pattern.
- `src/lib/integrations/ga4.ts` — OAuth + Data API pull filtered to the AI-referrer list.
- `src/lib/visibility/ai-referrers.ts` — the referrer domain list (single source of truth).
- `src/db/schema/visibility.ts` — `traffic_snapshots` (columns in `00-principles.md`).
- `src/app/api/visibility/traffic/route.ts` — GET series for the charts.
- `src/components/visibility/proof-panel.tsx` — the `/visibility` "Proof" section: score
  trend overlaid with clicks; AI-referral bars per engine; empty state = connect buttons
  with a one-line reason to connect each.
- Daily sync: extend the existing cron job — pull yesterday's rows for connected properties.

## Build steps

1. GSC OAuth + property selection (handle the "site not verified in GSC" case with a
   plain-language explainer, not an error).
2. Daily sync job, idempotent by (property, date); backfill 90 days on first connect so the
   chart isn't empty.
3. GA4 connect + AI-referrer filter; store per-engine counts in the same table.
4. Proof panel on `/visibility` per the section-loading convention; annotate the traffic
   chart with audit dates ("audit ran here") so deltas read causally.
5. Feed the numbers into V6.1 reports and the V8.5 digest ("clicks +23% since baseline;
   214 visits from ChatGPT this month").
6. Never meter any of this; connecting integrations costs nothing.
