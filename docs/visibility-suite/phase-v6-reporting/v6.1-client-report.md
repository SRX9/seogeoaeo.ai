- **Status:** ☐ Not started
- **Type:** ⚙️ Deterministic assembly
- **Pillar:** ⚪ Cross-cutting · Catalog #28
- **Depends on:** V2.3 + the analyzers (V2–V5)
- **Unlocks:** V6.2 (PDF), dashboard report view

## Goal

Aggregate all sub-scores + findings into one **business-language** report (in-app view + Markdown
export): dashboard, per-pillar sections, prioritized plan, and conservative traffic/revenue impact.

## Files to create

- `src/lib/visibility/report.ts` — `buildReport(auditId): ReportModel` + `toMarkdown(model)`.
- `src/app/(dashboard)/visibility/[auditId]/page.tsx` — report view.

## Build steps

1. Load the audit + sub-scores + findings + per-page rows from the DB (V0.3 tables).
2. Assemble the **12 sections**: executive summary, score dashboard, AI-visibility dashboard
   (per platform), crawler access table, brand authority, citability top-5/bottom-5, technical
   health, schema status, llms.txt status, prioritized action plan (quick wins/medium/strategic),
   competitor comparison (if run), glossary.
3. Translate technical findings into **owner/marketer language** with conservative traffic/revenue
   framing tied to score improvements.
4. Render in-app (React) + a Markdown export. Target 3,000–6,000 words, self-contained.
5. Test: report model has all sections + reflects stored scores.
