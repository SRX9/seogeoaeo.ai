# V6.2 — PDF report generator

- **Status:** ☐ Not started
- **Type:** ⚙️ Deterministic
- **Pillar:** ⚪ Cross-cutting · Catalog #29
- **Depends on:** [V6.1](v6.1-client-report.md)
- **Unlocks:** emailable deliverable, agency tier

## Goal

Render the report as a polished, branded **PDF** — cover with score gauge, color-coded score
tables, platform-readiness bar charts, severity findings, action plan, methodology appendix.

## Files to create

- `src/lib/visibility/report-pdf.ts` — `renderReportPdf(model, brand?): Uint8Array`.
- `src/app/api/visibility/[auditId]/pdf/route.ts` — download endpoint.
- print stylesheet (port the CSS) under `src/styles/visibility-report.css`.

## Build steps

1. **Do not port ReportLab.** Reuse the report HTML (V6.1) + a print stylesheet → PDF via Cloudflare
   **Browser Rendering** (headless Chromium) in the Worker, or a print-to-PDF path.
2. Port the **layout + color rules**: cover (navy gradient, score badge, brand/domain/date), score
   tables auto-colored green/blue/amber/red, severity callout blocks, section page breaks, styled
   JSON code blocks.
3. **Gauge thresholds:** green 80+ / blue 60–79 / yellow 40–59 / red <40.
4. Accept an optional brand config (V7.4 white-label) for colors/name/logo.
5. Test: PDF generates for a sample audit; gauge color matches score band.

## 📐 Build from (source of truth)

- `inspiration-code/scripts/generate_pdf_report.py` — the **section order, gauge color thresholds,
  and chart set** (spec only — don't run ReportLab).
- `inspiration-code/templates/geo-report-style.css` + `geo-report-template.html` — **reuse directly**
  as our HTML/CSS (cover layout, color-coding rules, navy/coral palette).
