## Goal

Run the audit/brand/citability stack against named competitors → a **side-by-side gap table**
(who owns which platform, who has the Wikipedia entity, who's more citable).

Ships in two stages: **lite** right after V2.3 — technical + citability vs one competitor,
because "you 68, them 81" makes the score meaningful long before the V5 depth exists — then
the **full grid** (brand, platforms, entity presence) once V5 lands. V5.5's answer-run data,
where it exists, adds "who actually gets named by the engines" columns to the grid.

## Files to create

- `src/lib/visibility/competitors.ts` — `benchmarkCompetitors(siteId, competitorUrls): CompareGrid`.
- a comparison dashboard view.

## Build steps

1. Read competitors from the brand profile (content-roadmap Phase 2) or accept ad-hoc URLs.
2. Run `runAudit` (or a lighter subset: technical + citability + brand + platforms) for each
   competitor; reuse all V2–V5 modules.
3. Build a comparison grid: overall + each sub-score + per-platform readiness + Wikipedia/entity
   presence + top-citability, you vs each competitor, with gap highlights.
4. Generate "catch-up" actions where a competitor leads (e.g., "Competitor X has a Wikipedia entity;
   you don't — pursue notability").
5. Test: a 2-competitor fixture renders a grid + gap callouts.
