02 — System architecture

One worker, many bindings

A request never leaves the edge for app logic. The Worker hosts the whole Next.js app; data goes through Hyperdrive to Postgres, ephemeral caches live in KV, and outbound work (email, LLM calls, search, publishing) fans out to bound services and external APIs. A separate agent-workflow Worker runs the long-lived daily automation as a durable Cloudflare Workflow.

04 — The visibility engine

Deterministic scorers, dual-mode

Every analyzer is a pure function ported 1:1 from a reference implementation and locked by golden tests. Each exposes one run(input) → { score, findings, data } contract, so the exact same code powers a full audit, a single Toolbox tool, and the in-editor draft score.

Citability×0.25

Passage-level quotability for AI answers; page = avg of top-5 blocks.

Brand×0.20

Off-site entity strength — Wikipedia/Wikidata, Reddit, YouTube, LinkedIn.

E-E-A-T×0.20

Experience/expertise signals + deterministic AI-content red-flag scan.

Technical×0.15

9-category health: SSR-heaviest, meta, crawl, security, CWV, mobile, URL.

Schema×0.10

JSON-LD/Microdata/RDFa detect + validate + point-table score + generator.

Platform×0.10

Presence & synergy across 5 answer/assistant engines.

Feeds the AI-visibility sub-score

05 — The math

How a number gets made

Two independent composites. The overall score reflects classic + answer + AI health; the AI-visibility score isolates findability inside assistants. Both are pure functions of the sub-scores.

Overall composite

overall = citability·0.25 + brand·0.20 + eeat·0.20
+ technical·0.15 + schema·0.10 + platform·0.10
citability .25
brand .20
eeat .20
technical .15
schema .10
platform .10

Bands: 90 / 75 / 60 / 40. Missing analyzer ⇒ 0 (partial audits still score).

AI-visibility sub-composite

aiVisibility = citability·0.35 + brand·0.30
+ crawler·0.25 + llmstxt·0.10
citability .35
brand .30
crawler .25
llmstxt .10

The proof stack the product retains on: score delta → answer share → real GSC/GA4 traffic.
